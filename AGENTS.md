# AGENTS.md

## 项目目标

本项目用于研究一款音乐游戏的本地歌曲/谱面文件结构，并辅助制作自制谱面。目标是：

1. 分析参考曲文件夹中的文件结构、文件类型、命名规则和可能的用途。
2. 记录歌曲格式、谱面格式、音频格式、替换流程和测试方法。
3. 编写 Python 工具，用于扫描、解析、验证和辅助生成自制谱面。
4. 所有改动必须在本项目目录内完成，不得直接修改原始游戏目录。

## 重要限制

- 不要修改 `/mnt/c` 或 `/mnt/e` 下的原始游戏文件。
- 不要删除、覆盖、重命名原始文件。
- 所有测试替换都应在备份副本或独立测试目录中完成。
- 修改代码前先说明计划；修改后说明改了哪些文件、如何验证。

## 工作语言

请使用中文回答。文件名、命令、报错信息保留原文。

## 推荐工作流程

1. 先扫描 reference 目录，生成文件清单。
2. 推测每类文件用途，并写入 docs/file_inventory.md。
3. 编写 scripts/inspect_folder.py，输出文件大小、扩展名、magic bytes、hash。
4. 如果发现疑似谱面文件，只读分析其结构，并把发现写入 docs/format_notes.md。
5. 编写 validate_song.py，检查一个自制歌曲文件夹是否包含必要文件。
6. 在 work/custom_song_001 中制作自制内容，禁止直接改 reference。
7. 每一步都尽量生成可复现的脚本，而不是手动操作。

## 完成标准

一个任务完成时，需要满足：

- 文档已更新；
- 脚本能在 WSL 中运行；
- 没有修改 reference 原始样本，除非用户明确同意；
- 给出下一步建议；# AGENTS.md
